# Publish CdA Calculator suite

The combined GitHub repository is:

- Repository: https://github.com/hkmsn/CDA_CALCULATOR
- Site: https://hkmsn.github.io/CDA_CALCULATOR/
- Local checkout: `/Users/marknelson/Documents/CDA_CALCULATOR`

Source projects:

- Monkey C: `/Users/marknelson/Documents/dev/MONKEY_C/CdA_Calculator`
- ESP32: `/Users/marknelson/Documents/dev/ESP32/PlatformIO/Projects/ESP32_AERO`

## Initial setup

Run once, only if the combined checkout does not already exist:

```bash
cd /Users/marknelson/Documents
git clone https://github.com/hkmsn/CDA_CALCULATOR.git
```

## Update the combined repository

Start from a clean combined checkout and update it without rewriting history:

```bash
cd /Users/marknelson/Documents/CDA_CALCULATOR
git pull --ff-only
```

Preview both synchronizations before applying them:

```bash
rsync -avni --delete \
  --exclude='.git/' \
  --exclude='.DS_Store' \
  --exclude='developer_key' \
  --exclude='bin/' \
  --exclude='~$*' \
  --exclude='__pycache__/' \
  --exclude='*.pyc' \
  /Users/marknelson/Documents/dev/MONKEY_C/CdA_Calculator/ \
  /Users/marknelson/Documents/CDA_CALCULATOR/CdA_Calculator/

rsync -avni --delete \
  --exclude='.git/' \
  --exclude='.pio/' \
  --exclude='venv/' \
  --exclude='.DS_Store' \
  --exclude='__pycache__/' \
  --exclude='*.pyc' \
  --exclude='compile_commands.json' \
  /Users/marknelson/Documents/dev/ESP32/PlatformIO/Projects/ESP32_AERO/ \
  /Users/marknelson/Documents/CDA_CALCULATOR/ESP32_AERO/
```

After reviewing the dry-run output, apply the updates by removing `n` from
`-avni` in both commands. Then update the top-level documentation:

```bash
cp /Users/marknelson/Documents/dev/MONKEY_C/CdA_Calculator/AeroDoc.html \
  /Users/marknelson/Documents/CDA_CALCULATOR/AeroDoc.html
```

Review and publish only the intended targets:

```bash
cd /Users/marknelson/Documents/CDA_CALCULATOR
git status --short
git diff --stat
git add AeroDoc.html CdA_Calculator ESP32_AERO
git diff --cached --stat
git commit -m "Update calculator, firmware, and documentation"
git push
```

Do not use `git push --force` for routine updates. Do not use `git add .` when
the checkout contains unrelated changes.

pandoc doc/AeroDoc.docx -o output_file.md
